# CourtLens 서비스 기획안

## 1. 기본 정보

| 항목 | 내용 |
| --- | --- |
| 서비스 명칭 | CourtLens |
| 한 줄 소개 | AI 하이라이트와 직관메이트로 팬 경험을 만들고, 그 반응을 구단의 이벤트·스폰서·ROI 운영툴로 바꾸는 농구 팬덤 플랫폼 |
| 1차 적용 종목 | 농구, KBL 중심 |
| 1차 MVP | 팬 앱 프로토타입 + 액션형 구단 대시보드 |
| 핵심 수익 모델 | 구단 B2B SaaS, 구축비, 성과형 수수료, 스폰서 패키지 수익 쉐어 |
| 확장 방향 | KBL 구단 파일럿 이후 축구, 배구, 야구 등 프로스포츠로 확장 |

## 2. 기획 배경

한국 프로스포츠는 팬덤의 열기는 크지만 팬이 보는 세계와 구단이 운영하는 세계 사이의 거리가 크다. 라이트 팬은 경기를 보면서도 어떤 선수를 봐야 하는지, 방금 플레이가 왜 좋은지, 다음 경기를 왜 봐야 하는지 이해하기 어렵다. 혼자 직관을 가는 것도 부담이다.

반대로 구단은 팬이 어떤 선수와 장면에 반응하는지, 어떤 좌석과 굿즈를 원하는지, 어떤 이벤트가 실제 예매나 스폰서 가치로 이어질지 빠르게 파악하기 어렵다. 팬 반응은 SNS, 커뮤니티, 유튜브 댓글에 흩어져 있고 이를 단기 이벤트나 스폰서 제안서로 바꾸는 일은 수작업에 가깝다.

CourtLens는 이 간극을 줄인다. 팬에게는 경기 전에는 직관메이트 탐색을, 경기 후에는 경기 summary, 오늘 잘한 선수, 출전 선수 기반 하이라이트 메이커, 쉬운 해설을 제공한다. 구단에게는 직관메이트 행동, 하이라이트 생성 행동, 팬 댓글 반응을 분석하는 데서 끝나지 않고 이벤트 실행, 스폰서 상품화, 성과 추적까지 연결하는 액션형 B2B SaaS를 제공한다.

## 3. 핵심 문제

### 팬 문제

- 전체 경기를 볼 시간은 부족하지만 좋아하는 선수의 장면만 빠르게 보기 어렵다.
- 농구 전술과 선수 역할을 모르면 좋은 장면을 놓친다.
- 기존 스포츠 앱은 일정, 결과, 기록, 뉴스 중심이라 라이트 팬이 계속 사용할 이유가 약하다.
- 처음 직관하거나 혼자 직관하는 팬은 좌석, 응원 문화, 동행에 부담을 느낀다.

### 구단 문제

- 팬들이 어떤 선수, 장면, 콘텐츠에 반응하는지 실시간으로 보기 어렵다.
- 팬 의견이 흩어져 있어 좌석, 굿즈, 경기장 경험, 선수 콘텐츠 니즈를 운영 액션으로 바꾸기 어렵다.
- 시즌 중 바로 실행할 단기 이벤트와 라운드 종료 후 정기 이벤트를 데이터 기반으로 기획하기 어렵다.
- 프랜차이즈 스타, 복귀 선수, 기록 달성 선수 같은 장기 마케팅 자산을 체계적으로 관리하기 어렵다.
- 스폰서에게 팬 몰입 장면과 반응 근거를 설득력 있게 제시하기 어렵다.

## 4. 제품 가설

1. 경기 전 팬은 `이 경기를 누구와 보러 갈지`를 먼저 해결하고 싶어 한다.
2. 경기 후 라이트 팬은 `오늘 어떤 경기에서 누가 잘했는지`를 먼저 알고 싶어 한다.
3. 경기 summary와 오늘의 스타 플레이어를 본 뒤, 해당 경기를 뛴 선수 중 보고 싶은 선수를 고르는 흐름이 자연스럽다.
4. 하이라이트를 이해한 직후가 입덕패스, 댓글, 예매/굿즈/이벤트 CTA로 연결하기 가장 좋은 순간이다.
5. 구단은 팬 반응이 이벤트 브리프와 스폰서 제안서로 바로 변환된다면 SaaS 비용을 지불할 이유가 있다.
6. CourtLens의 핵심 자산은 영상 자체가 아니라 `팬 행동 데이터 → 실행 가능한 매출 액션`으로 이어지는 운영 흐름이다.

## 5. 서비스 구조

CourtLens는 팬 앱과 구단 대시보드로 구성된다.

### 팬 앱

팬 앱은 팬이 실제로 사용하는 프론트 스테이지다. 목표는 라이트 팬이 경기를 더 쉽게 이해하고 다음 행동으로 넘어가게 만드는 것이다.

핵심 기능:

- 팀 선택과 KBL 공식 시즌 일정 기반 달력형 경기 선택
- 경기가 없는 날짜 disabled 처리
- 선택 경기 summary와 오늘 잘한 선수 노출
- 하이라이트 탭 내부 하이라이트 메이커에서 해당 경기 출전 선수 선택
- 미출전 선수 흐림 처리와 선택 비활성화
- 선택 경기/출전 선수 기반 AI 하이라이트 생성 인터랙션
- 작전판AI 초보/농잘알 해설
- 입덕패스: 경기 summary, 선수 관전 포인트, 팬 댓글
- 직관메이트 모집글 작성 및 댓글형 참여

팬 앱에서 쌓이는 핵심 행동 데이터:

- 하이라이트 조회, 저장, 공유
- 선택 경기, 선택 선수, 생성 클립
- 미출전 선수 비활성화 상태에서의 탐색/이탈
- 작전판AI 열람
- 입덕패스 summary 조회와 댓글 작성/반응
- 경기 전 직관메이트 클릭, 모집글 작성, 댓글 참여
- 예매/굿즈/이벤트 CTA 클릭

### 작전판AI 상세 기획

작전판AI는 하이라이트를 단순히 보여주는 기능이 아니라, 라이트 팬이 "방금 왜 좋은 장면이었는지" 이해하게 만드는 해설 레이어다. 하이라이트 카드에서 특정 클립을 선택하면 같은 장면을 두 가지 난이도로 설명한다.

핵심 역할:

- 초보 팬에게 장면의 의미를 쉬운 말로 설명한다.
- 농구를 아는 팬에게는 전술, 포지션, 수비 로테이션, 패스 선택을 설명한다.
- 선택 경기, 선택 선수, 장면 유형을 기반으로 설명을 개인화한다.
- 설명 열람 로그를 구단 대시보드의 Fan Insight와 Performance 입력값으로 넘긴다.
- 저장/공유/입덕패스 댓글/예매 CTA로 이어지는 다음 행동을 만든다.

화면 구성:

- 선택 클립 제목과 관련 선수
- 초보 모드 설명: "농구를 몰라도 이해되는 한 문장"
- 농잘알 모드 설명: 전술 키워드와 장면 맥락
- 장면 태그: 3점, 속공, 골밑, 어시스트, 수비, 클러치 등
- 다음 행동 CTA: 입덕패스 보기, 댓글 남기기, 직관메이트 찾기, 예매/굿즈 CTA

데이터 연결:

- 입력: 경기 ID, 클립 ID, 선수 ID, 장면 유형, 박스스코어, play-by-play 이벤트, 선택 팬 세그먼트
- 출력: 쉬운 해설, 전술 해설, 장면 태그, 추천 다음 행동
- 로그: 작전판AI 열람, 모드 전환, CTA 클릭, 설명 저장/공유

구단 가치:

- 구단은 어떤 장면이 단순 조회를 넘어 "이해"와 "다음 행동"으로 이어졌는지 볼 수 있다.
- 작전판AI 열람률이 높은 장면은 입덕패스 콘텐츠, 선수 콘텐츠, 스폰서 해설 상품으로 재활용할 수 있다.
- 예: "작전판AI 전술 해설 스폰서", "오늘의 플레이 해설 presented by 브랜드".

### 입덕패스 상세 기획

입덕패스는 경기 후 처음 들어온 팬이 해당 경기와 팀을 빠르게 이해하도록 돕는 팬 온보딩 카드다. 하이라이트가 장면 중심이라면, 입덕패스는 경기와 선수 맥락을 묶어 "이 팀을 왜 계속 봐야 하는지"를 설명한다.

핵심 역할:

- 선택 경기 summary를 짧고 명확하게 제공한다.
- 오늘 잘한 선수와 이유를 보여준다.
- 선수별 입덕 포인트를 정리한다.
- 팬 댓글과 반응을 모아 구단 대시보드의 Fan Voice 입력값으로 넘긴다.
- 다음 경기 직관, 선수 굿즈, 이벤트 참여로 이어지는 연결 지점을 만든다.

화면 구성:

- 경기 summary: 경기 흐름, 승부처, 팀 특징
- 오늘 잘한 선수: 기록, 역할, 왜 봐야 하는지
- 선수별 입덕 포인트: 슈터, 리바운더, 패서, 수비수 등 팬이 이해할 수 있는 캐릭터
- 댓글 리스트: 팬이 느낀 장면, 질문, 굿즈/이벤트 요청
- 반응 버튼: 공감, 더 보고 싶어요, 굿즈 원해요, 다음 경기 가고 싶어요
- CTA: 하이라이트 생성, 작전판AI 보기, 직관메이트 찾기, 이벤트 참여

데이터 연결:

- 입력: 경기 summary, 스타 플레이어, 하이라이트 태그, 작전판AI 설명, 팬 댓글
- 출력: 입덕 카드 문구, 선수 관전 포인트, 댓글/반응 카테고리
- 로그: summary 열람, 선수 카드 열람, 댓글 작성, 댓글 공감, CTA 클릭

구단 가치:

- 입덕패스는 라이트 팬의 첫 관심을 댓글, 굿즈 니즈, 이벤트 니즈로 수집하는 지점이다.
- 댓글 반응은 Fan Voice에서 좌석, 굿즈, 경기장 경험, 선수 콘텐츠, 이벤트, 응원 문화로 분류된다.
- 예: "신인 선수 굿즈 요청 증가 → 한정 포토카드 테스트", "초보 팬 질문 증가 → 입덕패스 경기 프리뷰 이벤트".

### 구단 대시보드

구단 대시보드는 팬 앱에서 발생한 행동을 수익화와 운영 개선으로 바꾸는 백오피스다. 최신 방향성은 `분석 화면`이 아니라 `매출 액션 운영툴`이다.

핵심 메뉴:

1. **Fan Insight**
   - 경기별/선수별 팬 반응, 하이라이트 저장/공유, 작전판AI 열람, 댓글 반응, 직관메이트 클릭을 분석한다.
   - Fan Voice와 Player Context DB를 결합해 팬의 세계와 선수 맥락을 함께 본다.

2. **Revenue Playbook**
   - 이번 주 매출 기회 Top 3를 제시한다.
   - 단기 이벤트, 라운드 이벤트, 장기 프랜차이즈 마케팅 후보를 우선순위화한다.

3. **AI Strategy Engine**
   - 팬 행동과 선수 맥락을 이벤트 브리프, 스폰서 제안서, Performance 개선안으로 변환한다.
   - 구단 담당자가 바로 검토할 수 있도록 대상, 문구, 노출 위치, KPI, 근거 지표를 구조화한다.

4. **Event Builder**
   - 추천 액션을 실제 팬 앱 이벤트 카드 초안으로 만든다.
   - 대상 팬 세그먼트, 이벤트명, CTA 문구, 노출 위치, 운영 기간을 수정할 수 있다.

5. **Sponsor Package Builder**
   - 인기 장면과 팬 반응 지표를 스폰서 영업 제안서 초안으로 묶는다.
   - 예: 오늘의 클러치 플레이, 라운드 MVP 팬 투표, 직관메이트 매칭 이벤트 스폰서.

6. **Performance / ROI**
   - 하이라이트 생성부터 예매 CTA, 굿즈 CTA, 이벤트 참여, 스폰서 이벤트 반응까지 추적한다.
   - 각 단계별 이탈률, 전환율, 관련 선수, 예상 매출 기여, 개선 액션을 제공한다.

7. **Fan Voice / Matchday Ops**
   - 좌석, 굿즈, 경기장 경험, 선수 콘텐츠, 이벤트, 응원 문화 의견을 분류한다.
   - 운영 개선 리포트 또는 이벤트 테스트로 연결한다.

## 6. AI Strategy Engine

AI Strategy Engine은 CourtLens의 B2B 설득 포인트다. 단순히 "팬 반응이 높습니다"를 보여주는 것이 아니라, "그래서 이번 주에 무엇을 실행해야 하는가"를 구단에게 제공한다.

### 입력 데이터

- 팬 행동: 조회, 저장, 공유, 댓글, 작전판AI 열람, 직관메이트 클릭, CTA 클릭
- 경기 맥락: 경기 결과, 상대팀, 홈/원정, 경기 summary, 오늘 잘한 선수
- 선수 맥락: 포지션, 최근 폼, 해당 경기 출전 여부, 출전 시간, 기록 달성, 복귀/부상, 프랜차이즈 스타 여부
- 경기/운영 맥락: 다음 홈/원정 경기, 좌석, 굿즈 재고, 라운드 일정, 이벤트 캘린더
- 스폰서 제약: 카테고리, 브랜드 톤, 노출 위치, 금지 문구
- Fan Voice: 좌석, 굿즈, 경기장 경험, 선수 콘텐츠, 이벤트, 응원 문화 의견

### 처리 방식

1. 대량 로그와 팬 의견을 분류한다.
2. 선수/장면/콘텐츠별 매출 기회 후보를 점수화한다.
3. 예상 효과와 실행 난이도를 계산한다.
4. Top 3 기회를 Revenue Playbook으로 제시한다.
5. Event Builder와 Sponsor Package Builder에 들어갈 구조화 초안을 만든다.
6. Performance 결과를 다시 다음 추천에 반영한다.

### 산출물

- 이번 주 매출 기회 Top 3
- 이벤트 브리프
- 팬 앱 이벤트 카드 초안
- 스폰서 제안서 초안
- Performance 개선 액션
- 예상 매출 기여와 KPI

### 기획 기준 모델 스택

- 전략 생성/이벤트 기획: `gpt-5.5`
- 작전판AI/입덕패스 문구 생성: `gpt-5.5`
- 대량 분류/세그먼트 라벨링: `gpt-5.4-mini`
- 저비용 후보 스코어링/랭킹: `gpt-5.4-nano`
- 중계 ASR/자막 처리: `gpt-4o-transcribe` 또는 `gpt-4o-mini-transcribe`
- 영상 클립 추출: OpenCV, 경기 시간 OCR, play-by-play 동기화

운영 원칙:

- AI가 자동 게시하지 않는다.
- 구단 담당자 승인 후 팬 앱에 노출한다.
- 추천마다 근거 지표와 관련 선수/콘텐츠/팬 세그먼트를 붙인다.
- 출력은 자유 문장이 아니라 구조화 JSON으로 저장할 수 있게 설계한다.

## 7. MVP 정의

아이디어톤 MVP는 완전한 영상 분석 모델이 아니다. 목표는 팬 앱과 구단 대시보드가 하나의 데이터 흐름으로 연결되고, 이 흐름이 B2B 수익화로 이어질 수 있음을 보여주는 것이다.

MVP 정의:

> KBL 공식 시즌 일정 기반 팬 경험 프로토타입과 액션형 구단 대시보드를 통해, 선택 경기의 하이라이트·입덕패스·직관메이트 반응이 이벤트, 스폰서 제안, ROI 추적으로 이어지는 흐름을 시연한다.

### MVP 포함 범위

- 창원 LG 세이커스 / 수원 KT 소닉붐 팀 전환
- KBL 공식 API 기반 정적 선수/기록/시즌 일정 데이터
- 달력형 경기 선택과 경기 없는 날짜 disabled 처리
- 선택 경기 summary와 오늘 잘한 선수 노출
- 하이라이트 탭 내부 선수 선택
- 해당 경기를 뛰지 않은 선수 흐림 처리와 선택 비활성화
- 선택 경기/출전 선수 기반 AI 하이라이트 생성 인터랙션
- 작전판AI 초보/농잘알 해설과 열람 로그
- 입덕패스 경기 summary, 선수 관전 포인트, 댓글 반응
- 직관메이트 모집글 작성 및 댓글형 참여
- 구단 대시보드 Top 3 Revenue Opportunity
- Fan Insight / Revenue Playbook / Performance 3축
- AI Strategy Engine 설명 카드와 모달
- Event Builder, Sponsor Package Builder, Fan Voice, Matchday Ops, Performance 모달
- 생성 결과함 문서 저장

### MVP 제외 범위

- 실제 영상 AI 분석 모델 완성
- 실제 경기 영상 자동 업로드/처리
- 실제 play-by-play와 영상 타임라인 자동 매칭
- 실제 결제, 예매, 굿즈몰, CRM 연동
- 구단 관리자 계정/권한 시스템
- 실시간 경기 중 자동 하이라이트 생성

## 8. MVP 데모 시나리오

### 데모 1: 경기 전 팬이 직관메이트를 찾는다

1. 팬이 선호팀을 선택한다.
2. 보고 싶은 경기를 선택한다.
3. 경기 전 상태에서는 직관메이트 창으로 들어간다.
4. 같은 경기를 보러 갈 직관메이트를 찾는다.
5. 좌석, 응원 성향, 관심 선수를 기준으로 모집글을 작성하거나 댓글로 참여한다.

### 데모 2: 경기 후 팬이 하이라이트를 이해한다

1. 팬이 선호팀을 선택한다.
2. 다시 볼 경기를 선택한다.
3. 경기 summary와 오늘 잘한 선수가 먼저 보인다.
4. 하이라이트 메이커에서 해당 경기를 뛴 선수만 선택한다.
5. 뛰지 않은 선수는 흐림 처리되어 선택할 수 없다.
6. AI 하이라이트 카드가 생성된다.
7. 작전판AI가 장면을 쉬운 말과 전술 언어로 설명한다.
8. 팬은 입덕패스에서 경기 summary, 선수 관전 포인트, 팬 댓글을 확인한다.

### 데모 3: 구단이 팬 반응을 이벤트로 전환

1. 구단 대시보드 첫 화면에서 이번 주 매출 기회 Top 3를 확인한다.
2. 예: 유기상 3점 하이라이트 반응 급상승 → 응원석 단기 이벤트 추천.
3. Event Builder에서 대상, 문구, 노출 위치, 운영 기간을 수정한다.
4. 생성 결과함에 이벤트 초안을 저장한다.
5. Performance에서 예매 CTA 클릭과 예상 매출 기여를 확인한다.

### 데모 4: 구단이 스폰서 제안서를 만든다

1. 인기 장면 또는 Top 3 카드에서 Sponsor Package Builder를 연다.
2. 대표 장면, 팬 반응 지표, 예상 노출 위치, 제안 문구를 확인한다.
3. 스폰서 제안서를 생성 결과함에 저장한다.
4. 구단은 이 문서를 스폰서 영업 자료 초안으로 활용한다.

## 9. 비즈니스 모델

### 구단 SaaS 사용료

구단이 월 단위 또는 시즌 단위로 대시보드 사용료를 낸다.

제공 가치:

- 선수/장면별 팬 반응 분석
- Revenue Playbook
- Event Builder
- Sponsor Package Builder
- Performance 리포트
- 시즌별 팬덤/매출 기회 리포트

구단별 앱, 티켓, 굿즈, CRM, 브랜딩 환경에 맞춰 세팅하는 비용을 받는다.

- 구단별 컬러/문구/템플릿 적용
- 티켓/굿즈 링크 연동
- 관리자 권한 세팅
- 이벤트/스폰서 제안서 템플릿 세팅

CourtLens 이벤트와 CTA를 통해 발생한 전환에 대해 일부 수수료를 받는다.

- 예매 CTA 클릭
- 굿즈 CTA 클릭
- 이벤트 참여

인기 장면과 팬 반응 데이터를 기반으로 스폰서 상품을 구성하고, 구단과 플랫폼이 수익을 공유한다.

## 10. 데이터 구조

MVP에서는 실제 서버 대신 정적 JS 데이터와 샘플 행동 데이터를 사용한다.

핵심 엔티티:

- Team: 구단명, 약칭, 홈구장, 컬러
- Player: 선수명, 등번호, 포지션, 기록, 입덕 포인트
- Game: 경기 날짜, 상대팀, 홈/원정, 상태, 경기 summary, 오늘 잘한 선수
- GamePlayer: 경기 ID, 선수 ID, 출전 여부, 출전 시간, 주요 기록, 선택 가능 여부
- HighlightClip: 경기 ID, 장면명, 관련 선수, 쉬운 해설, 전술 해설
- FanComment: 경기 ID, 하이라이트 ID, 작성 내용, 감정 라벨, 운영 카테고리
- FanAction: view, save, share, click, comment
- TacticBoardInsight: 경기 ID, 클립 ID, 쉬운 해설, 전술 해설, 장면 태그, 다음 행동
- RookiePass: 경기 ID, summary, 오늘 잘한 선수, 입덕 포인트, 댓글/반응 요약
- MatePost: 경기, 좌석, 관심 선수, 응원 성향, 댓글형 참여
- RevenueOpportunity: 장면, 반응 근거, 예상 효과, 실행 난이도, 추천 액션
- GeneratedAsset: 이벤트 문서, 스폰서 제안서, Performance 리포트

제품화 단계에서는 play-by-play 이벤트, 영상 타임코드, 팬 세그먼트, CRM 연동 데이터를 추가한다.

## 11. 성공 기준

팬 앱:

- 사용자가 하이라이트, 작전판AI, 입덕패스, 직관메이트의 역할을 구분할 수 있다.
- 사용자가 달력에서 경기가 있는 날짜만 선택할 수 있다.
- 하이라이트 메이커는 하이라이트 탭 안에서만 보인다.
- 초보 팬이 "이 경기에서 오늘 잘한 선수는 이 선수였고, 이 장면을 보면 되겠다"는 판단을 할 수 있다.
- 하이라이트 메이커에서 출전 선수와 미출전 선수를 직관적으로 구분할 수 있다.
- 직관메이트 행동과 하이라이트/입덕패스 반응이 구단 대시보드 데이터로 이어지는 구조가 보인다.

구단 대시보드:

- 첫 화면에서 "그래서 무엇을 실행할 수 있는가"가 보인다.
- Top 3 기회가 이벤트 또는 스폰서 제안서로 이어진다.
- Performance에서 예매, 굿즈, 이벤트, 스폰서 반응을 확인할 수 있다.
- B2B SaaS 비용을 낼 이유가 `실행 문서 생성`과 `성과 추적`으로 설명된다.

## 13. 한 문장 정리

CourtLens는 팬에게는 경기 summary와 출전 선수 중심 하이라이트로 경기를 이해하고 함께 보러 갈 이유를 만들며, 구단에게는 하이라이트 생성 행동과 팬 댓글 반응을 이벤트, 스폰서, ROI로 전환하는 액션형 스포츠 팬덤 SaaS다.
