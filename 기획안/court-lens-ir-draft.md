# CourtLens IR Draft

## 1. Core Concept

프로젝트명:

코트렌즈(CourtLens)

주제 문장:

> AI로 스포츠 진입장벽을 낮추면, 팬은 경기를 더 재밌게 보고 구단은 팬을 더 오래 볼 수 있다.

한 줄 소개:

> CourtLens는 KBL 팬이 팀과 경기를 고르면 경기 summary와 오늘 잘한 선수를 보여주고, 해당 경기 출전 선수 기반 AI 하이라이트와 쉬운 전술 해설을 구단의 이벤트·스폰서·ROI 운영으로 연결하는 농구 팬 경험 플랫폼입니다.

핵심 포지셔닝:

- 팬 앱은 라이트 팬이 농구를 쉽게 이해하도록 경기 summary, 오늘 잘한 선수, 출전 선수 기반 하이라이트를 제공한다.
- 직관메이트는 하이라이트 관심을 다음 경기 동행과 직관 의도로 연결한다.
- 구단 대시보드는 팬 반응을 보여주는 리포트가 아니라 이벤트 실행, 스폰서 상품화, 성과 추적까지 이어지는 액션형 B2B SaaS다.

핵심 메시지:

> CourtLens 구단 대시보드는 팬 반응을 보여주는 데서 끝나지 않고, 콘텐츠 생성, 이벤트 실행, 스폰서 상품화, 전환 성과 추적까지 연결하는 액션형 B2B SaaS입니다.

## 2. Problem

프로농구는 재미없는 스포츠가 아니다. 문제는 처음 보는 팬이 재미를 느끼기까지 넘어야 할 장벽이 높고, 구단은 팬 관심을 실제 행동과 매출로 연결하기 어렵다는 점이다.

팬의 문제:

- 전체 경기를 볼 시간은 부족하지만 좋아하는 선수의 장면만 빠르게 모아보기 어렵다.
- 픽앤롤, 스페이싱, 미스매치 같은 전술을 모르면 좋은 플레이를 놓친다.
- 방금 나온 플레이가 왜 좋은지 설명해주는 서비스가 부족하다.
- 혼자 직관을 가기 부담스럽고 같은 관심사의 팬을 찾기 어렵다.
- 기존 스포츠 앱은 일정, 결과, 기록, 뉴스 중심이라 라이트 팬이 계속 사용할 이유가 약하다.

구단의 문제:

- 팬들이 어떤 선수, 장면, 콘텐츠에 반응하는지 실시간으로 알기 어렵다.
- SNS, 커뮤니티, 유튜브 댓글에 흩어진 팬 반응을 이벤트/굿즈 기획으로 바꾸기 어렵다.
- 시즌 중 바로 실행할 단기 이벤트와 라운드 종료 정기 이벤트를 데이터 기반으로 만들기 어렵다.
- 프랜차이즈 스타, 복귀 선수, 기록 달성 선수처럼 장기 마케팅할 자산을 체계적으로 관리하기 어렵다.
- 스폰서에게 "우리 팬들이 무엇에 몰입하는지"를 데이터로 설득하기 어렵다.

문제 정의:

> 팬에게는 경기 summary와 출전 선수 중심으로 경기를 쉽게 이해하고 다음 행동까지 이어가는 경험이 필요하고, 구단에게는 팬의 세계를 읽어 단기 이벤트, 라운드 이벤트, 장기 프랜차이즈 마케팅으로 바꿔주는 도구가 필요하다.

## 3. Insight

1. 팬은 기록표보다 내가 좋아하는 선수의 장면과 그 의미에 반응한다.
2. 라이트 팬은 장면을 이해할수록 선수를 더 기억하고, 경기를 더 자주 본다.
3. 하이라이트를 본 직후가 티켓, 굿즈, 직관메이트, 이벤트로 연결하기 가장 좋은 순간이다.
4. 구단에게 필요한 것은 분석 리포트가 아니라 바로 실행할 이벤트와 스폰서 제안서다.
5. 팬 반응은 단기 이벤트, 라운드 이벤트, 장기 프랜차이즈 마케팅의 입력값이 될 수 있다.

선순환:

> 팬이 팀과 경기를 고른다 → 경기 summary와 오늘 잘한 선수를 확인한다 → 하이라이트 메이커에서 해당 경기를 뛴 선수만 고른다 → AI가 개인화 하이라이트와 쉬운 해설을 만든다 → 팬이 입덕패스에 댓글 반응을 남긴다 → 직관메이트/예매/굿즈로 이어진다 → 구단은 팬 관심과 댓글 반응 데이터를 얻는다 → AI Strategy Engine이 이벤트와 스폰서 제안서로 바꾼다 → Performance로 성과를 추적한다

## 4. Product Structure

CourtLens는 팬 앱과 구단 대시보드로 구성된다.

### A. 팬 앱

팬 앱의 목적은 팬에게 명확한 결과를 주는 것이다. 팬은 앱을 켰을 때 "내 팀에서 오늘 누구를 보면 되는지", "내가 고른 선수의 장면이 무엇인지", "방금 플레이가 왜 좋은지", "다음 직관을 누구와 갈 수 있는지"를 바로 알아야 한다.

핵심 기능:

1. 팀/경기 선택
   - 창원 LG 세이커스와 수원 KT 소닉붐 전환
   - 팀별 경기 카드
   - 경기 선택 시 경기 summary와 오늘 잘한 선수 표시

2. 하이라이트 메이커
   - 선택 경기 기준 선수 칩
   - 해당 경기를 뛴 선수만 선택 가능
   - 뛰지 않은 선수는 흐림 처리와 선택 비활성화
   - 선택한 출전 선수 조합으로 개인화 하이라이트 카드 생성
   - MVP에서는 실제 영상 생성이 아니라 인터랙션 목업
   - 실제 제품에서는 play-by-play, OCR/ASR, 영상 타임코드를 연결

3. 작전판AI
   - 초보 모드: 쉬운 말로 장면 설명
   - 농잘알 모드: 전술 맥락 설명

4. 입덕패스
   - 선택 경기 summary 제공
   - 오늘 잘한 선수와 이유 제공
   - 선수와 팀의 관전 포인트 제공
   - 팬 댓글 작성과 반응 수집
   - 라이트 팬이 "오늘 이 선수를 보면 되는 이유"를 이해

5. 직관메이트
   - 관심 선수, 경기, 좌석, 응원 성향 기반 모집글 작성
   - 홈/원정 경기 선택
   - 신청 멘트 입력
   - 혼직관 부담 완화와 구단의 직관 의도 데이터 확보

### B. 구단 대시보드

구단 대시보드는 `Fan Insight → Revenue Playbook → AI Strategy Engine → Event Builder/Sponsor Package → Performance` 흐름으로 작동한다.

1. Fan Insight
   - 경기별/선수별 팬 반응, 하이라이트 조회/저장/공유, 입덕패스 댓글, 직관메이트 클릭, Fan Voice, Player Context DB를 묶어 팬 경험을 분석한다.

2. Revenue Playbook
   - 이번 주 매출 기회 Top 3를 제시한다.
   - 단기 이벤트, 라운드 이벤트, 장기 프랜차이즈 마케팅을 추천한다.

3. AI Strategy Engine
   - 팬 행동과 선수 맥락을 이벤트 브리프, 스폰서 제안서, Performance 개선안으로 변환한다.
   - 대상, 문구, 노출 위치, 예상 KPI, 근거 지표를 구조화한다.

4. Event Builder
   - 추천 액션을 팬 앱 이벤트 카드 초안으로 만든다.
   - 대상 팬 세그먼트, 이벤트명, CTA 문구, 노출 위치, 기간을 수정한다.

5. Sponsor Package Builder
   - 인기 장면과 팬 반응 데이터를 스폰서 영업 제안서 초안으로 묶는다.

6. Performance / ROI
   - 경기 선택, 하이라이트 메이커 선수 선택, 하이라이트 생성, 작전판AI 열람, 입덕패스 댓글, 직관메이트 클릭, 예매 CTA, 굿즈 CTA, 스폰서 이벤트 참여를 추적한다.
   - 단계별 이탈률, 전환율, 관련 선수, 예상 매출 기여, 개선 액션을 제공한다.

## 6. MVP Scope

현재 MVP는 KBL 두 팀을 기준으로 한다.

대상 팀:

- 창원 LG 세이커스
- 수원 KT 소닉붐

MVP의 목적:

> 완전한 영상 분석 모델을 만드는 것이 아니라, 팬 앱에서 발생한 관심이 구단 대시보드에서 매출 액션으로 바뀌는 흐름을 증명한다.

MVP 핵심 데모:

1. 팬이 선호팀과 경기를 고른다.
2. 경기 summary와 오늘 잘한 선수를 확인한다.
3. 하이라이트 메이커에서 해당 경기를 뛴 선수만 고른다.
4. 개인화 하이라이트 카드가 생성된다.
5. 작전판AI가 장면을 쉽게 설명한다.
6. 팬은 입덕패스에 댓글 반응을 남기고 직관메이트로 다음 경기 행동을 만든다.
7. 구단은 Top 3 매출 기회를 확인한다.
8. Event Builder로 이벤트 초안을 만든다.
9. Sponsor Package Builder로 스폰서 제안서를 만든다.
10. Performance에서 예매, 굿즈, 이벤트, 스폰서 반응을 추적한다.

## 7. AI Strategy and Model Decision

CourtLens는 하나의 모델에 모든 일을 맡기지 않는다. 업무 성격에 따라 모델을 나눈다.

1. 전략 생성/이벤트 기획
   - 모델: `gpt-5.5`
   - 용도: Revenue Playbook, Event Builder, Sponsor Package Builder, Performance 개선안 생성
   - 이유: 구단 마케팅 전략은 선수 맥락, 팬 반응, 경기 일정, 스폰서 제약을 함께 고려해야 하므로 높은 추론 품질과 문안 품질이 중요하다.

2. 대량 분류/세그먼트 라벨링
   - 모델: `gpt-5.4-mini`
   - 용도: Fan Voice 분류, 행동 로그 요약, 선수/장면별 이벤트 후보 태깅, CTA 반응 분류
   - 이유: 호출량이 많고 지연·비용 민감도가 높다.

3. 저비용 스코어링/랭킹
   - 모델: `gpt-5.4-nano`
   - 용도: 후보 사전 필터링, 간단 태그, 매출 기회 점수 초안
   - 이유: 모든 후보를 상위 모델로 보내지 않고 후보군을 먼저 줄인다.

4. 음성/자막 처리
   - 모델: `gpt-4o-transcribe` 또는 `gpt-4o-mini-transcribe`
   - 용도: 중계 음성 ASR, 자막 추출, 경기 시간 동기화 보조

5. 영상 처리
   - 기술: OpenCV, 경기 시간 OCR, play-by-play 동기화
   - 용도: 이벤트 앞뒤 8~12초 클립 추출, 선수/장면 태깅 보조

운영 원칙:

- MVP와 IR에서는 `GPT-5.6` preview 접근성을 전제로 삼지 않는다.
- 출력은 자유 문장이 아니라 JSON 형태의 구조화 출력으로 받는다.
- 추천 근거는 RAG로 가져온 구단 데이터, 선수 맥락, 팬 행동 지표에 연결한다.
- 팬 앱 게시와 스폰서 제안서 발송은 반드시 구단 담당자 승인 단계를 거친다.

## 8. Business Model

팬에게 직접 과금하는 모델은 핵심이 아니다. CourtLens의 수익 모델은 구단과 스폰서 중심이다.

### 구단 SaaS 사용료

- Fan Insight
- Revenue Playbook
- AI Strategy Engine
- Event Builder
- Sponsor Package Builder
- Performance / ROI
- 시즌별 팬덤/매출 기회 리포트

### 초기 구축비

- 구단별 브랜딩 적용
- 티켓/굿즈 링크 연동
- 관리자 계정 설정
- 이벤트 템플릿 세팅
- 스폰서 제안서 템플릿 세팅

### 성과형 수수료

- 하이라이트 시청 후 홈경기 예매
- 선수 콘텐츠 시청 후 유니폼 구매
- 단기 이벤트 참여
- 티켓 CTA 클릭
- 굿즈 테스트 클릭
- 스폰서 이벤트 참여

### 스폰서 패키지 수익 쉐어

- 오늘의 클러치 플레이 스폰서
- 작전판AI 전술 해설 스폰서
- 직관메이트 매칭 이벤트 스폰서
- 라운드 MVP 팬 투표 스폰서
- 프랜차이즈 스타 굿즈 스폰서

## 9. Competitive Difference

기존 KBL/스포츠 앱:

- 일정, 결과, 기록, 뉴스 중심
- 팬이 경기의 의미를 이해하도록 돕는 설명이 부족함
- 좋아하는 선수 조합으로 직접 하이라이트를 만들기 어려움
- 직관 동행/관전 포인트를 하이라이트와 연결하지 못함

유튜브/숏폼 하이라이트:

- 편집자 기준의 공통 하이라이트
- 선수별/취향별 개인화 부족
- 구단이 팬 반응을 직접 소유하기 어려움
- 시청 후 티켓/굿즈/스폰서 전환 흐름이 약함

SNS/커뮤니티:

- 팬 목소리는 많지만 흩어져 있음
- 구단이 일일이 모으고 해석해야 함
- 직관 동행 모집은 가능하지만 경기 데이터/하이라이트와 연결되지 않음

CourtLens:

- 팬이 직접 고른 선수 기준 AI 하이라이트 생성
